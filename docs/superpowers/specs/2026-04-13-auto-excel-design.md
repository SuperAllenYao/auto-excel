# Auto Excel - 设计文档

## 概述

一个 macOS 专用的 CLI 工具，用于自动处理营销分析 Excel 文件。用户将原始 Excel 文件放入桌面指定文件夹，在终端执行一条命令即可全自动完成数据计算、排序、分组统计，并输出处理后的新文件。

## 技术栈

| 层级 | 选型 | 版本 |
|------|------|------|
| 语言 | Python | 3.11+ |
| 包管理 | uv | 0.11+ |
| Excel 处理 | openpyxl | 3.1.5+ |
| 数据处理 | pandas | 2.0+ |
| CLI 框架 | Typer | 0.9+ |
| 终端输出 | rich | 14.1+ |

## 目标用户

小团队内部分发，成员有基本技术背景（能执行终端命令）。

## 一、安装流程

### 用户操作

一行命令完成全部安装：

```bash
curl -sSL https://raw.githubusercontent.com/<repo>/main/install.sh | bash
```

### install.sh 自动执行内容

1. 将项目 clone 到 `~/.auto-excel/`
2. 检测 uv 是否已安装，若未安装则通过 `curl -LsSf https://astral.sh/uv/install.sh | sh` 自动安装
3. 进入项目目录执行 `uv sync`，创建虚拟环境并安装全部 Python 依赖
4. 在 `~/.local/bin/` 下创建 `auto-excel` 包装脚本：
   ```bash
   #!/bin/bash
   cd ~/.auto-excel && uv run auto-excel "$@"
   ```
5. 确保 `~/.local/bin` 在用户 PATH 中（若不在则追加到 `~/.zshrc`）
6. 创建桌面工作目录：
   - `~/Desktop/marketing analysis/Raw/`
   - `~/Desktop/marketing analysis/New/`
   - `~/Desktop/marketing analysis/log/`
7. 输出安装成功提示

### 安装后使用

新开终端窗口，直接输入：

```bash
auto-excel on
```

## 四、核心处理流程

### 触发方式

用户在终端输入 `auto-excel on`，程序启动后全自动执行以下流程。

### 4.1 启动阶段

1. 扫描 `Raw/` 目录下所有 `.xlsx` 文件
2. 读取 `log/processed.json`，过滤掉已处理文件
3. 将待处理文件按创建时间从早到晚排序
4. 若无待处理文件，提示用户并退出

### 4.2 串行处理每个文件

对每个待处理文件依次执行以下三个流程（不并行）：

#### 流程 1：列计算

1. 将 Raw/ 中的原始文件复制到 New/（同名），后续所有操作在 New/ 的副本上进行
2. 打开 New/ 中的副本，定位到第 4 个 Sheet
3. 扫描第 1 行表头，按名称定位各列位置
4. 依次插入 4 个计算列（每次插入前重新扫描表头以应对列偏移）：

| 新列名 | 插入位置 | 计算公式 | 备注 |
|--------|---------|---------|------|
| 实际花费 | "花费"列右侧 | 花费 ÷ 1.136 | 逐行计算，跳过第 1 行 |
| 点击率 | "点击量"列右侧 | 点击量 ÷ 展现量 | 行业标准 CTR |
| CPC | "点击率"列右侧 | 实际花费 ÷ 点击量 | 每次点击成本 |
| 实际成本 | "留资成本"列右侧 | 实际花费 ÷ 留资人数 | 每次留资成本 |

**零值保护：** 当分母为 0 时，单元格填入 0，同时在日志中记录该异常行号。

#### 流程 2：排序

- 按"实际成本"列数据从高到低排序
- 第 1 行（表头）固定不动，仅排序第 2 行及以下的数据行

#### 流程 3：分组统计

1. 在"互动成本"列右侧插入一列，第 1 行填入表头"占比"
2. 将数据按"实际成本"值分为三个区间：

| 区间 | 条件 | 操作 |
|------|------|------|
| 高成本 | 实际成本 ≥ 90 | 合并该区间在"占比"列的单元格 |
| 中成本 | 50 ≤ 实际成本 < 90 | 合并该区间在"占比"列的单元格 |
| 低成本 | 实际成本 < 50 | 合并该区间在"占比"列的单元格 |

3. 在每个合并后的单元格中填入：`条数/占比`（如 `15/30%`）
   - 条数 = 该区间的数据行数
   - 占比 = 该区间条数 ÷ 总数据行数

> 注：因流程 2 已按实际成本从高到低排序，同一区间的行是连续的，可以直接合并。

### 4.3 收尾

- 保存文件
- 将该文件写入 `processed.json`（标记为已处理）
- 处理失败的文件不写入 `processed.json`，下次运行会重新处理

### 4.4 结束阶段

- 所有文件处理完毕后，输出本次执行报告（表格形式）
- 写入日志文件
- 退出程序，杀掉进程

## 六、状态管理

### processed.json

**路径：** `~/Desktop/marketing analysis/log/processed.json`

**格式：**
```json
{
  "campaign_report_Q1.xlsx": {
    "processed_at": "2026-04-13T15:30:00",
    "status": "success"
  }
}
```

**规则：**
- 程序启动时读取此文件，Raw/ 中不在此文件中的 .xlsx 文件视为"未处理"
- 处理成功后立即写入（每处理完一个文件就更新一次，而非全部完成后批量写入）
- 处理失败的文件不写入，下次运行会自动重试

---

## 七、日志文件

**路径：** `~/Desktop/marketing analysis/log/YYYY-MM-DD.log`

**内容：**
- 每次运行的启动/结束时间
- 每个文件的处理详情（各步骤耗时）
- 异常记录（如除以零的行号、找不到列名等）
- 错误堆栈信息（仅日志文件中记录，终端不显示）

## 八、错误处理策略

| 错误场景 | 处理方式 |
|---------|---------|
| Raw/ 目录为空或无待处理文件 | 友好提示，正常退出 |
| Excel 文件没有第 4 个 Sheet | 跳过该文件，标记失败，继续处理下一个 |
| 找不到指定列名（如"花费"） | 跳过该文件，标记失败，继续处理下一个 |
| 除法分母为 0 | 单元格填入 0，日志记录异常行号 |
| 文件被其他程序占用 | 跳过该文件，标记失败，继续处理下一个 |
| New/ 中已存在同名文件 | 覆盖写入 |
